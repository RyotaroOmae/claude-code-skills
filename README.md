# claude-code-skills

[Claude Code](https://claude.com/claude-code) 用に自作した **Skills**（スラッシュコマンド／自動起動スキル）と **Subagents**（サブエージェント定義）をまとめたリポジトリです。

気象シミュレーション [SCALE](https://scale.riken.jp/) / LES の解析ワークフロー（富岳・gale などのサーバー、NetCDF、PBS/PJM バッチジョブ）を中心に、コードレビューや日々の開発作業を効率化するためのものが多く含まれます。

## 使い方

`skills/` と `agents/` の各ディレクトリを、Claude Code が読み込む設定ディレクトリに配置（またはシンボリックリンク）します。

```bash
# 例: ユーザーグローバルに配置する場合
ln -s "$(pwd)/skills"  ~/.claude/skills
ln -s "$(pwd)/agents"  ~/.claude/agents
```

配置後、スキルは `/skill-name` で呼び出せます。サブエージェントは Claude が必要に応じて自動的に、または明示的な指示で起動します。

## Skills 一覧

### レビュー・コード品質

| Skill | 概要 |
| --- | --- |
| `adversarial-review` | correctness-reviewer を2体立て、独立にレビューさせて互いの指摘を反証し合わせ、生き残った指摘だけを統合報告する 2体特化の adversarial review。 |
| `team-review` | Agent teams で複数のレビュアーに多観点（バグ・目的整合・性能・科学的正しさ）で並行検証させ、反証し合わせてから統合報告するチームレビュー。 |
| `codex-review` | Codex (MCP) にコードレビューさせる。引数なしなら現ブランチの差分、引数ありでその対象をレビュー。 |
| `careful-code-editing` | コードの実装・修正・リファクタ・テスト追加時の行動ルール。早合点・過剰設計・範囲外変更・検証不足を避けるチェックリスト。 |
| `explain-code` | Claude が作成/編集したコードや既存コードを、責任者であるユーザーが中身まで理解できる形で解説（最小コードへの蒸留・理解度クイズ付き）。 |
| `explain-diff` | 既知の「アンカー」との差分だけを解説し、学習の転移で素早く腹落ちさせる。追加/変更/削除のデルタと「なぜ・挙動がどう変わるか」を説明し差分クイズで定着確認。ゼロから全部なら `explain-code`。 |

### Git・安全な操作

| Skill | 概要 |
| --- | --- |
| `commit` | コミットメッセージ規約（何を・なぜ変えたかを具体的に）に従って git コミットを作成。秘密情報チェックと main/master 誤コミット防止付き。 |
| `careful` | 慎重モード。`rm -rf`・force push・`reset --hard`・上書き・大量削除など不可逆操作の前に、正確なコマンドを提示して明示確認を取る。 |

### SCALE / 気象解析ワークフロー

| Skill | 概要 |
| --- | --- |
| `ncinspect` | NetCDF ファイルの中身（次元・変数・属性・座標範囲・基本統計）を素早く要約。解析・プロット前の中身確認に。 |
| `jobwatch` | PBS バッチジョブを `qstat` で確認し、実験ログから ERROR/WARNING/NaN を検出・要約。富岳(PJM)ログ診断モードも持つ。 |
| `scpwatch` | scp/rsync 転送中ファイルの完了を監視し、完了後に指定の後続処理（プロット・解析）を自動起動。 |

### 汎用・Claude Code 設定

| Skill | 概要 |
| --- | --- |
| `codex` | Codex (MCP) になんでも依頼する汎用窓口。自然言語の指示をそのまま実行させる。 |
| `codewiki` | リポジトリの CodeWiki 形式ドキュメント（アーキテクチャ・モジュール・フロー）を生成／更新。 |
| `claude-docs` | Claude Code の設定・機能を公式ドキュメントで裏取りする。架空の設定キーをでっち上げないために使う。 |

## Subagents 一覧

| Agent | 概要 |
| --- | --- |
| `correctness-reviewer` | バグと「本来の目的・要求どおりに動くか」を調べるレビュアー。ロジック誤り・エッジケース漏れ・要求との不整合を重点的に見る。 |
| `perf-reviewer` | 性能観点のレビュアー。計算量・無駄な処理・I/O やメモリ・ホットパスのボトルネックを調べる。 |
| `science-reviewer` | 解析コードの科学的正しさを検証。単位・物理的整合・数式・統計手法・座標/格子や NetCDF の扱いを重点的に見る。 |
