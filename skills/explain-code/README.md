# explain-code

Claude Code が書いた変更、または指定された既存コードを、**責任者であるユーザーが
中身まで理解・承認できる**形に整えて解説するスキル。ファイル一覧・目的・コードの流れに
加え、**使っているツール/ライブラリの役割**・**本質を残した最小コードへの蒸留**・
**理解度クイズ**まで行う。コメント補完やドキュメント生成は、ユーザーの同意/要望時のみ。

> 説明タスクは `model: sonnet` で動く（軽量モデルで十分なため）。この上書きは
> このターン限りで、次の発話で元のモデルに自動で戻る。

## 使い方

- **自動参照**: コードの実装・修正を一通り終えた場面など、`description` に
  合致するときに Claude Code が参照する。
- **明示呼び出し**: `/explain-code` で実行。説明対象を引数で絞れる
  （例: `/explain-code src/foo.py`）。引数が無ければ直前の会話で
  編集したファイルを対象にする。引数で既存ファイルを指定した場合は、
  「変更」ではなく「既存コードの理解」モードで解説する。

## 何をするか

1. 対象ファイルを実際に読み、**ファイル一覧 / 目的 / コードの流れ / 確認すべき点**
   をチャットに整理して出す（変更モード/理解モードで見出しを切り替える）。
2. **依存ツール・ライブラリの役割**を解説する（numpy/xarray/MPI/NetCDF など、
   その文脈で何をしているか・特有の作法がどこで効くか）。
3. **本質でない防御的処理（ファイル無しの分岐・try/except・引数パース等）を削いだ
   最小コード**を対比提示し、「結局このコードは何をしているか」を蒸留する
   （説明用の蒸留であり、実ファイルは書き換えない）。
4. 最後に**理解度クイズ（3〜5問）**を出し、責任者の理解を確認する（解答を待つ）。

## 任意（ユーザーの同意/要望時のみ）

- 意図が追えない箇所への**解説コメント追記**：候補を提示し、同意を得てから追記する。
- 変更解説を `CHANGES_<日付>.md` として作業ディレクトリに残す（望まれたときのみ。
  クイズは問題のみ記載し、模範解答はネタバレ防止で書かない）。

## しないこと

- コードの機能変更（ロジック修正が要るときは報告で指摘するだけ）。
- 最小コードを実ファイルへ反映すること（あくまで説明用の蒸留）。
- ユーザーの同意なしに実ファイルを書き換えること（コメント追記・ドキュメント生成）。
- 推測での説明・コメント（読み取れない箇所は「要確認」として挙げる）。

詳細は `SKILL.md` を参照。
