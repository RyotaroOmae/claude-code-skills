# explain-diff

既に理解している**アンカー(基準)との差分(デルタ)だけ**を解説して、対象コードを
**学習の転移 (transfer of learning)** で素早く腹落ちさせるスキル。共有部分は
意図的に省き、追加/変更/削除のデルタを既知の言葉に接続して説明し、最後に
**差分クイズ**で定着を確認する。

> 説明タスクは `model: sonnet` で動く（軽量モデルで十分なため）。この上書きは
> このターン限りで、次の発話で元のモデルに自動で戻る。

## 使い方

- **自動参照**: 「○○との違いだけ教えて」「△△は分かってる前提で」など、
  `description` に合致するときに Claude Code が参照する。
- **明示呼び出し**: `/explain-diff [説明対象] [基準/アンカー]` で実行。
  - 例: `/explain-diff plot_new.py plot_mynn.py`（plot_mynn.py との差分で説明）。
  - 対象を省くと直前の会話で編集したコードが対象。
  - 基準(アンカー)を省くと、Claude が似た既存ファイル・定番ライブラリ・直前に
    説明したコードから候補を推測し、**確認を取ってから**差分を説明する
    （有効なアンカーが見つからなければ指定を依頼するか explain-code を案内する）。

## 何をするか

1. **対象とアンカーを確定**する（ユーザー明示があればそれ、無ければ推測→確認）。
2. アンカーと対象の**共有部分を箇条書きし「ここは既知として省略する」と宣言**する。
3. 差分を **追加/変更/削除** で構造化し、各デルタを
   「アンカーでは X、ここでは Y。**なぜ Y か・挙動がどう変わるか**」で解説する
   （依存ライブラリは差分に関わる部分だけ）。
4. 差分を**既知の言葉に接続**し、メンタルモデルをどう拡張すれば理解できるかを示す。
5. 最後に**差分クイズ（3〜5問）**を出し、理解を確認する（解答を待つ）。

## explain-code との使い分け

| | explain-diff | explain-code |
|---|---|---|
| 発想 | 既知との**差分だけ**を解説（共有部分は省く） | **ゼロから自己完結で全部**解説 |
| 向く場面 | 派生スクリプト、おなじみ手法の新用途、「違いだけ知りたい」 | 単独の新規コード、「全部腹落ちさせたい」 |
| 出力 | 共有部分の省略宣言 / デルタ / アンカー接続 / 差分クイズ | ファイル一覧 / 流れ / 依存解説 / 最小蒸留 / クイズ |
| ファイル変更 | しない（理解支援に専念） | 任意でコメント追記・`CHANGES_*.md`（同意時のみ） |

## しないこと

- 共有部分・既知部分の**冗長な再解説**（それが要るなら explain-code）。
- **実ファイルの書き換え**（コメント追記・ドキュメント生成はしない）。
- 推測での説明（差分・アンカーが読み取れない箇所は「要確認」とする）。
- コードの機能変更（ロジック修正が要るときは報告で指摘するだけ）。

詳細は `SKILL.md` を参照。
