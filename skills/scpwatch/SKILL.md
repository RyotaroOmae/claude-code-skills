# /scpwatch — 転送完了の監視と後続処理の自動実行

Arguments passed: `$ARGUMENTS`

## 引数の解釈

- `--` より前: 監視対象のファイルパス（複数・グロブ可）。
- `--` より後: 完了後に行う作業（コマンドでも自然言語の指示でもよい）。
- 引数が曖昧なら、直近の会話文脈から「どのファイルを待っているか」「終わったら何を
  するつもりだったか」を補う。

## 完了判定

転送中ファイルは「存在するがまだ書き込まれている」状態なので、以下で判定する:

1. **サイズ安定法（基本）**: `stat -c %s` を 30〜60 秒間隔で取り、**2回連続で同サイズ**
   かつサイズ > 0 なら完了とみなす。
2. 期待サイズが会話で分かっている場合（例:「最終サイズは 57G」）はそれとの一致を優先。
3. `lsof <file>` が使えるなら、書き込みプロセスが消えたことも補助判定に使う。
4. scp が `qsub scp.sh` でジョブ化されている場合は `qstat` の終了も併用する（/jobwatch 参照）。

## 監視の実行

ポーリングは `run_in_background` の Bash で行う（フォアグラウンドで sleep し続けない）:

```bash
while true; do
  s1=$(stat -c %s <file> 2>/dev/null || echo 0)
  sleep 45
  s2=$(stat -c %s <file> 2>/dev/null || echo 0)
  [ "$s1" = "$s2" ] && [ "$s2" -gt 0 ] && break
done
echo "TRANSFER_COMPLETE: <file> ($s2 bytes)"
```

複数ファイルの場合は全ファイルの安定を待つ。ただし「最初の1つが終わり次第着手」と
指示されたらファイルごとに後続処理を始めてよい。

## 完了後

1. 完了したファイルのサイズと `ncdump -h`（NetCDF の場合）で中身を簡易確認する。
2. `--` 以降に指定された後続処理を実行する。
3. NetCDF の重い処理になる場合は、サーバー負荷を考慮して計算ノードへのジョブ投入も検討する。

## Gotchas

- 転送中のファイルを読むと壊れた途中状態を解析してしまう。**必ず完了判定を通してから**処理する。
- rsync は一時ファイル名（`.file.XXXX`）で書いてから rename するため、対象ファイル自体が
  現れた時点でほぼ完了していることがある。`ls -la` で隠し一時ファイルの有無も見る。
- 監視間隔は転送速度に合わせる。数十 GB 級なら 45〜60 秒で十分。
